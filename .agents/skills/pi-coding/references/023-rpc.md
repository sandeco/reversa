# RPC Mode

RPC mode enables headless operation via JSON protocol over stdin/stdout. Use it to embed the agent in other applications, IDEs, or custom UIs.

> [!note]
> For Node.js/TypeScript, use `AgentSession` directly from `@mariozechner/pi-coding-agent` instead of spawning a subprocess. See [[023-rpc#sdk-reference|sdk reference]] for the API, or [`src/modes/rpc/rpc-client.ts`](../src/modes/rpc/rpc-client.ts) for a subprocess-based client.

## Starting RPC Mode

```bash
pi --mode rpc [options]
```

| Option | Description |
|--------|-------------|
| `--provider <name>` | LLM provider (anthropic, openai, google, etc.) |
| `--model <pattern>` | Model pattern or ID (`provider/id:<thinking>`) |
| `--no-session` | Disable session persistence |
| `--session-dir <path>` | Custom session storage directory |

## Protocol Overview

- **Commands**: JSON objects to stdin, one per line
- **Responses**: JSON with `type: "response"` indicating success/failure
- **Events**: Agent events streamed to stdout as JSON lines

All commands support optional `id` for request/response correlation.

### Framing

Strict JSONL with LF (`\n`) as the only record delimiter.

> [!warning]
> Node `readline` is **not** protocol-compliant â€” it splits on Unicode separators (`U+2028`, `U+2029`) which are valid inside JSON strings. Clients must split on `\n` only, stripping optional trailing `\r`.

## Commands

### Prompting

#### prompt

Send a user prompt. Events continue streaming asynchronously after acceptance.

```json
{"id": "req-1", "type": "prompt", "message": "Hello, world!"}
```

With images:
```json
{"type": "prompt", "message": "What's in this image?", "images": [{"type": "image", "data": "base64-encoded-data", "mimeType": "image/png"}]}
```

**During streaming** â€” must specify `streamingBehavior`:

| Value | Behavior |
|-------|----------|
| `"steer"` | Queue message; delivered after current turn finishes tool calls |
| `"followUp"` | Wait until agent stops; delivered when agent finishes |

If agent is streaming and no `streamingBehavior` is specified, returns an error.

- **Extension commands** (e.g., `/mycommand`): Execute immediately, even during streaming
- **Skill commands** (`/skill:name`) and **prompt templates** (`/template`): Expanded before sending/queueing

Response:
```json
{"id": "req-1", "type": "response", "command": "prompt", "success": true}
```

- `success: true` = prompt accepted, queued, or handled immediately
- `success: false` = prompt rejected before acceptance

#### steer

Queue a steering message while agent runs. Delivered after current turn finishes tool calls, before next LLM call.

```json
{"type": "steer", "message": "Stop and do this instead"}
```

With images:
```json
{"type": "steer", "message": "Look at this instead", "images": [{"type": "image", "data": "base64-encoded-data", "mimeType": "image/png"}]}
```

Skill commands and prompt templates are expanded. Extension commands are not allowed (use `prompt`).

Response:
```json
{"type": "response", "command": "steer", "success": true}
```

#### follow_up

Queue a follow-up message to be processed after agent finishes.

```json
{"type": "follow_up", "message": "After you're done, also do this"}
```

With images:
```json
{"type": "follow_up", "message": "Also check this image", "images": [{"type": "image", "data": "base64-encoded-data", "mimeType": "image/png"}]}
```

Delivered only when agent has no more tool calls or steering messages. Skill commands and prompt templates are expanded. Extension commands are not allowed.

Response:
```json
{"type": "response", "command": "follow_up", "success": true}
```

#### abort

Abort current agent operation.

```json
{"type": "abort"}
```

Response:
```json
{"type": "response", "command": "abort", "success": true}
```

#### new_session

Start a fresh session. Can be cancelled by a `session_before_switch` extension event handler.

```json
{"type": "new_session"}
```

With optional parent session tracking:
```json
{"type": "new_session", "parentSession": "/path/to/parent-session.jsonl"}
```

Response:
```json
{"type": "response", "command": "new_session", "success": true, "data": {"cancelled": false}}
```

`cancelled: true` if an extension cancelled the operation.

```json
{"type": "get_state"}
```

```json
  "command": "get_state",

  "data": {
    "model": {...},
    "thinkingLevel": "medium",
    "isStreaming": false,

    "autoCompactionEnabled": true,

  }

```

`model` is a full [Model](#model) object or `null`. `sessionName` is set via `set_session_name`.

```json
{"type": "get_messages"}
```

Response:
```json

  "type": "response",

  "data": {"messages": [...]}
}
```

Messages are `AgentMessage` objects (see [Message Types](#message-types)).

### Queue Modes

#### set_steering_mode

Control how steering messages are delivered.

```json
{"type": "set_steering_mode", "mode": "one-at-a-time"}
```

| Mode | Behavior |
|------|----------|
| `"all"` | Deliver all steering messages after current assistant turn finishes |
| `"one-at-a-time"` | Deliver one steering message per completed assistant turn (default) |

#### set_follow_up_mode

Control how follow-up messages are delivered.

```json
{"type": "set_follow_up_mode", "mode": "one-at-a-time"}
```

| Mode | Behavior |
|------|----------|
| `"all"` | Deliver all follow-up messages when agent finishes |
| `"one-at-a-time"` | Deliver one follow-up message per agent completion (default) |

### Bash

#### bash

Execute a shell command and add output to conversation context.

```json
{"type": "bash", "command": "ls -la"}
```

Response:
```json
{
  "type": "response",
  "command": "bash",
  "success": true,
  "data": {
    "output": "total 48\n...",
    "exitCode": 0,
    "cancelled": false,
    "truncated": false
  }
}
```

If output was truncated, includes `fullOutputPath`:
```json
{"type": "response", "command": "bash", "success": true, "data": {"output": "truncated...", "exitCode": 0, "cancelled": false, "truncated": true, "fullOutputPath": "/tmp/pi-bash-abc123.log"}}
```

> [!info]
> Bash output reaches the LLM on the **next prompt**, not immediately. Multiple bash commands execute before a prompt; all outputs are included in the next LLM context.

#### abort_bash

Abort a running bash command.

```json
{"type": "abort_bash"}
```

Response:
```json
{"type": "response", "command": "abort_bash", "success": true}
```

### Commands

#### get_commands

Get available commands (extension commands, prompt templates, and skills). Invoke via `prompt` with `/` prefix.

```json
{"type": "get_commands"}
```

Response:
```json
{
  "type": "response",
  "command": "get_commands",
  "success": true,
  "data": {
    "commands": [
      {"name": "session-name", "description": "Set or clear session name", "source": "extension", "path": "/home/user/.pi/agent/extensions/session.ts"},
      {"name": "fix-tests", "description": "Fix failing tests", "source": "prompt", "location": "project", "path": "/home/user/myproject/.pi/agent/prompts/fix-tests.md"},
      {"name": "skill:brave-search", "description": "Web search via Brave API", "source": "skill", "location": "user", "path": "/home/user/.pi/agent/skills/brave-search/SKILL.md"}
    ]
  }
}
```

| Field | Description |
|-------|-------------|
| `name` | Command name (invoke with `/name`) |
| `description` | Human-readable description (optional for extensions) |
| `source` | `"extension"` (via `pi.registerCommand()`), `"prompt"` (from `.md` file), `"skill"` (name prefixed with `skill:`) |
| `location` | `"user"` (`~/.pi/agent/`), `"project"` (`./.pi/agent/`), `"path"` (explicit path) |
| `path` | Absolute file path (optional) |

> [!note]
> Built-in TUI commands (`/settings`, `/hotkeys`, etc.) are not included. They are handled only in interactive mode.

## Events

Events are streamed to stdout as JSON lines during agent operation. Events do NOT include an `id` field (only responses do).

### Event Types

| Event | Description |
|-------|-------------|
| `agent_start` | Agent begins processing |
| `agent_end` | Agent completes (includes all generated messages) |
| `turn_start` | New turn begins |
| `turn_end` | Turn completes (includes assistant message and tool results) |
| `message_start` | Message begins |
| `message_update` | Streaming update (text/thinking/toolcall deltas) |
| `message_end` | Message completes |
| `tool_execution_start` | Tool begins execution |
| `tool_execution_update` | Tool execution progress (streaming output) |
| `tool_execution_end` | Tool completes |
| `queue_update` | Pending steering/follow-up queue changed |
| `compaction_start` | Compaction begins |
| `compaction_end` | Compaction completes |
| `auto_retry_start` | Auto-retry begins (after transient error) |
| `auto_retry_end` | Auto-retry completes (success or final failure) |
| `extension_error` | Extension threw an error |

### agent_start / agent_end

```json
{"type": "agent_start"}
{"type": "agent_end", "messages": [...]}
```

### turn_start / turn_end

A turn consists of one assistant response plus any resulting tool calls and results.

```json
{"type": "turn_start"}
{"type": "turn_end", "message": {...}, "toolResults": [...]}
```

### message_start / message_end / message_update

```json
{"type": "message_start", "message": {...}}
{"type": "message_end", "message": {...}}
{"type": "message_update", "message": {...}, "assistantMessageEvent": {...}}
```

The `assistantMessageEvent` delta types:

| Type | Description |
|------|-------------|
| `start` | Message generation started |
| `text_start` / `text_delta` / `text_end` | Text content block streaming |
| `thinking_start` / `thinking_delta` / `thinking_end` | Thinking block streaming |
| `toolcall_start` / `toolcall_delta` / `toolcall_end` | Tool call streaming (includes full `toolCall` object) |
| `done` | Message complete (reason: `"stop"`, `"length"`, `"toolUse"`) |
| `error` | Error occurred (reason: `"aborted"`, `"error"`) |

### tool_execution_start / update / end

```json
{"type": "tool_execution_start", "toolCallId": "call_abc123", "toolName": "bash", "args": {"command": "ls -la"}}
{"type": "tool_execution_update", "toolCallId": "call_abc123", "toolName": "bash", "partialResult": {"content": [{"type": "text", "text": "partial output so far..."}], "details": {...}}}
{"type": "tool_execution_end", "toolCallId": "call_abc123", "toolName": "bash", "result": {...}, "isError": false}
```

`tool_execution_update.partialResult` contains accumulated output (not just delta), allowing clients to replace display on each update.

### queue_update

```json
{"type": "queue_update", "steering": ["Focus on error handling"], "followUp": ["After that, summarize the result"]}
```

### compaction_start / compaction_end

```json
{"type": "compaction_start", "reason": "threshold"}
{"type": "compaction_end", "reason": "threshold", "result": {"summary": "...", "firstKeptEntryId": "abc123", "tokensBefore": 150000, "details": {}}, "aborted": false, "willRetry": false}
```

| Field | Description |
|-------|-------------|
| `reason` | `"manual"`, `"threshold"`, or `"overflow"` |
| `result` | Compaction summary, or `null` if failed/aborted |
| `aborted` | `true` if aborted |
| `willRetry` | `true` if overflow and compaction succeeded â€” agent will retry automatically |

### auto_retry_start / auto_retry_end

```json
{"type": "auto_retry_start", "attempt": 1, "maxAttempts": 3, "delayMs": 2000, "errorMessage": "529 {\"type\":\"error\",...}"}
{"type": "auto_retry_end", "success": true, "attempt": 2}
{"type": "auto_retry_end", "success": false, "attempt": 3, "finalError": "529 overloaded_error: Overloaded"}
```

### extension_error

```json
{"type": "extension_error", "extensionPath": "/path/to/extension.ts", "event": "tool_call", "error": "Error message..."}
```

## Extension UI Protocol

Extensions request user interaction via `ctx.ui.select()`, `ctx.ui.confirm()`, etc. In RPC mode, these use a request/response sub-protocol.

**Dialog methods** (`select`, `confirm`, `input`, `editor`): emit `extension_ui_request` on stdout, block until client sends `extension_ui_response` on stdin with matching `id`.

**Fire-and-forget methods** (`notify`, `setStatus`, `setWidget`, `setTitle`, `set_editor_text`): emit `extension_ui_request` on stdout, no response expected.

> [!note]
> `ctx.hasUI` is `true` in RPC mode because the sub-protocol provides dialog functionality.

### Unsupported methods

These require direct TUI access and are degraded:

| Method | Behavior |
|--------|----------|
| `custom()` | Returns `undefined` |
| `setWorkingMessage()`, `setWorkingIndicator()`, `setFooter()`, `setHeader()`, `setEditorComponent()`, `setToolsExpanded()` | No-ops |
| `getEditorText()` | Returns `""` |
| `getToolsExpanded()` | Returns `false` |
| `pasteToEditor()` | Delegates to `setEditorText()` |
| `getAllThemes()` | Returns `[]` |
| `getTheme()` | Returns `undefined` |
| `setTheme()` | Returns `{ success: false, error: "..." }` |

### Extension UI Requests (stdout)

All requests: `{"type": "extension_ui_request", "id": "uuid", "method": "...", ...}`

| Method | Fields | Response |
|--------|--------|----------|
| `select` | `title`, `options[]`, `timeout?` | `value` (selected option) or `cancelled: true` |
| `confirm` | `title`, `message`, `timeout?` | `confirmed: true/false` or `cancelled: true` |
| `input` | `title`, `placeholder?` | `value` (entered text) or `cancelled: true` |
| `editor` | `title`, `prefill?` | `value` (edited text) or `cancelled: true` |
| `notify` | `message`, `notifyType?` ("info" \| "warning" \| "error") | Fire-and-forget |
| `setStatus` | `statusKey`, `statusText` (or `undefined` to clear) | Fire-and-forget |
| `setWidget` | `widgetKey`, `widgetLines` (or `undefined` to clear), `widgetPlacement?` ("aboveEditor" \| "belowEditor") | Fire-and-forget |
| `setTitle` | `title` | Fire-and-forget |
| `set_editor_text` | `text` | Fire-and-forget |

### Extension UI Responses (stdin)

```json
{"type": "extension_ui_response", "id": "uuid", "value": "selected"}
{"type": "extension_ui_response", "id": "uuid", "confirmed": true}
{"type": "extension_ui_response", "id": "uuid", "cancelled": true}
```

## Error Handling

Failed commands return:

```json
{"type": "response", "command": "set_model", "success": false, "error": "Model not found: invalid/model"}
```

Parse errors:

```json
{"type": "response", "command": "parse", "success": false, "error": "Failed to parse command: Unexpected token..."}
```

## Examples

### Basic Client (Python)

```python
import subprocess
import json

proc = subprocess.Popen(
    ["pi", "--mode", "rpc", "--no-session"],
    stdin=subprocess.PIPE,
    stdout=subprocess.PIPE,
    text=True
)

def send(cmd):
    proc.stdin.write(json.dumps(cmd) + "\n")
    proc.stdin.flush()

def read_events():
    for line in proc.stdout:
        yield json.loads(line)

# Send prompt
send({"type": "prompt", "message": "Hello!"})

# Process events
for event in read_events():
    if event.get("type") == "message_update":
        delta = event.get("assistantMessageEvent", {})
        if delta.get("type") == "text_delta":
            print(delta["delta"], end="", flush=True)
    
    if event.get("type") == "agent_end":
        print()
        break
```

### Interactive Client (Node.js)

See [`src/modes/rpc/rpc-client.ts`](../src/modes/rpc/rpc-client.ts) for a typed client implementation. For extension UI handling, see [`examples/rpc-extension-ui.ts`](../examples/rpc-extension-ui.ts) paired with [`examples/extensions/rpc-demo.ts`](../examples/extensions/rpc-demo.ts).

```javascript
const { spawn } = require("child_process");
const { StringDecoder } = require("string_decoder");

const agent = spawn("pi", ["--mode", "rpc", "--no-session"]);

function attachJsonlReader(stream, onLine) {
    const decoder = new StringDecoder("utf8");
    let buffer = "";

    stream.on("data", (chunk) => {
        buffer += typeof chunk === "string" ? chunk : decoder.write(chunk);

        while (true) {
            const newlineIndex = buffer.indexOf("\n");
            if (newlineIndex === -1) break;

            let line = buffer.slice(0, newlineIndex);
            buffer = buffer.slice(newlineIndex + 1);
            if (line.endsWith("\r")) line = line.slice(0, -1);
            onLine(line);
        }
    });

    stream.on("end", () => {
        buffer += decoder.end();
        if (buffer.length > 0) {
            onLine(buffer.endsWith("\r") ? buffer.slice(0, -1) : buffer);
        }
    });
}

attachJsonlReader(agent.stdout, (line) => {
    const event = JSON.parse(line);

    if (event.type === "message_update") {
        const { assistantMessageEvent } = event;
        if (assistantMessageEvent.type === "text_delta") {
            process.stdout.write(assistantMessageEvent.delta);
        }
    }
});

// Send prompt
agent.stdin.write(JSON.stringify({ type: "prompt", message: "Hello" }) + "\n");

// Abort on Ctrl+C
process.on("SIGINT", () => {
    agent.stdin.write(JSON.stringify({ type: "abort" }) + "\n");
});
```
