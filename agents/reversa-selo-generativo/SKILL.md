# Generative Seal

Creates **unique and reproducible visual seals** for Reversa projects using p5.js. Each project gets its own generative artwork derived from a deterministic seed: the same seed always generates the same seal.

Inspired by the Art Blocks pattern (seeded randomness) and Anthropic's `algorithmic-art` skill, but with deliberately smaller scope: it produces **only the seal**, not a philosophical manifesto, not an exploration platform. It is a decorative identity element.

## When to use

- **Hero for `index.html`** in the mini-site generated by `/reversa-documentation`.
- **Mini seal in the header** of each mini-site page (reduced version).
- **Slide cover** in `deck.html`.
- **Visual identity** for any artifact generated by Reversa that needs a distinctive mark.

The skill is **lightweight by design**: it uses only p5.js, generates a single canvas, and exports as SVG or PNG. It has no exploration sidebar, no multiple modes, no mandatory animation (animation is optional).

## Principles

1. **Absolute reproducibility**: the same seed always generates the same seal.
2. **Limited palette**: 3 to 5 colors per seal, derived from the chosen visual style.
3. **Balanced composition**: recognizable central shape with auxiliary elements orbiting or composing around it.
4. **No text on the canvas**: the seal is purely visual; project name and title stay outside the canvas, in HTML.
5. **Size-adaptable**: the same canvas works at 64x64 (mini header seal) and 800x800 (large hero).
6. **No dependencies beyond p5.js**: no GSAP, dat.GUI, lodash. Only p5.

## Workflow

### 1. Receive the seed

The seed can come from:

- **Direct string**: user or agent provides a sha256 hash, project name, or any string.
- **File path**: user points to `.reversa/soul.md`, the skill computes the content `sha256`.
- **Fallback**: if nothing is provided, use the current timestamp `sha256` ("of the moment" seal, not reproducible).

```javascript
async function computeSeed(input) {
    const enc = new TextEncoder();
    const data = enc.encode(input);
    const hashBuffer = await crypto.subtle.digest("SHA-256", data);
    const hashArray = Array.from(new Uint8Array(hashBuffer));
    const hashHex = hashArray.map((b) => b.toString(16).padStart(2, "0")).join("");
    return hashHex;
}
```

The seed is a 64-character hexadecimal string. The skill extracts multiple numbers from it to feed different parameters.

### 2. Choose palette

The palette derives from the visual style chosen by the user of `/reversa-documentation`, or it can be provided directly.

Consult `references/PALETTE_BY_STYLE.md` for the full palette table by style. The 4 base palettes are:

| Style | Palette |
|--------|--------|
| `sober` | Neutral tones: grays, teal-blue, light terracotta, off-white |
| `premium` | Dark mode: deep black, gold, wine red, silver, midnight blue |
| `dense` | Saturated: orange, cyan, magenta, lime, indigo |
| `exploratory` | Luminous pastels: aurora pink, glacier cyan, mist lilac, light white |

The choice of which color is "central" and which are "auxiliary" comes from the seed.

### 3. Choose generative pattern

The skill has **5 established patterns**, each with a distinct look. The seed determines which pattern to use (first 2 hex digits modulo 5):

| Pattern | Appearance | When it fits |
|--------|-----------|----------------|
| `flow-field` | Perlin flow fields, curved organic strokes | `sober`, `exploratory` style |
| `particle-orbit` | Particles orbiting a center with trails | `premium`, `exploratory` style |
| `crystal-lattice` | Symmetric crystalline shape, clean geometry | `dense`, `sober` style |
| `wave-interference` | Moire-like interference patterns, circular waves | `premium`, `dense` style |
| `noise-strata` | Horizontal noise strata, abstract landscape | `sober`, `exploratory` style |

Details in `references/GENERATIVE_PATTERNS.md`.

The skill can accept a pattern override via parameter, ignoring seed derivation.

### 4. Generate the code

Structure of the resulting HTML:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Seal: <!-- PROJECT_NAME --></title>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.0/p5.min.js"></script>
    <style>
        body { margin: 0; display: flex; align-items: center; justify-content: center; min-height: 100vh; background: var(--seal-bg, #0a0a14); }
        #seal-container { display: block; }
        .label { color: var(--seal-fg, #eaeaea); text-align: center; margin-top: 16px; font-family: system-ui, sans-serif; }
    </style>
</head>
<body>
    <div>
        <div id="seal-container"></div>
        <div class="label"><!-- PROJECT_NAME --></div>
    </div>
    <script id="seal-config" type="application/json">
        <!-- CONFIG_JSON -->
    </script>
    <script>
        // 1. Read config (seed, palette, pattern, size)
        // 2. Seed p5.js: randomSeed(seedInt); noiseSeed(seedInt);
        // 3. Execute chosen pattern
        // 4. Save canvas as reference
    </script>
</body>
</html>
```

**Fundamental rules**:

1. **Standalone HTML**: single file, p5.js via CDN, nothing external.
2. **p5.js 1.9.0+**: use `https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.0/p5.min.js`.
3. **Always seeded**: `randomSeed(seedInt)` and `noiseSeed(seedInt)` in the first setup, before any `random()` or `noise()` call.
4. **Convert hex hash to int**: take the first 16 hex chars, convert via `parseInt(hex.slice(0, 16), 16)`.
5. **Square canvas**: default size 800x800, but parameterizable.
6. **`noLoop()`**: by default the seal is static. If animation is requested, use `frameRate(30)` and a low-cost algorithm.
7. **No text on the canvas**: project name stays in `<div class="label">` outside the canvas.

### 5. Size variants

The skill accepts a `size` parameter to generate a large seal (hero, 800x800), medium (slide cover, 400x400), or mini (header, 64x64 or 128x128).

For smaller seals (<200px), apply automatic simplifications:
- Reduce particle count by 80%.
- Increase stroke thickness proportionally.
- Disable animation.

### 6. Save and deliver

Output is standalone HTML. The canvas can be exported as PNG via button (optional) or as SVG for use in headers (recommended for mini seals: SVG scales perfectly).

## Quality guidelines

- **Recognizability**: the seal must have a strong central form; someone who saw it once should recognize the project by the seal.
- **Chromatic balance**: never use all palette colors equally in the same seal; typically 1 dominant color (60%), 1 secondary (30%), 1 accent (10%).
- **Do not draw more attention than the content**: the seal is identity, not focus. In a large hero, it is okay to be the protagonist; in a mini header seal, it should give space to the project name.
- **Accessibility**: minimum contrast between background and primary elements. Seals with dark background should have light elements and vice versa.
- **Language**: comments in English, without em dashes.

## Code guidelines

- **Named constants at the top**: size, palette, pattern parameters visible at the beginning.
- **Single generation function**: each pattern is a `drawFlowField()`, `drawCrystalLattice()`, etc. function, isolated and testable.
- **No global side effects**: everything scoped within p5 `setup()` and `draw()`.

Consult `references/ERRORS.md` for scenarios (p5.js unavailable, canvas unsupported, invalid seed, extreme size).
